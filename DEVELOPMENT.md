# Live2D MOD 开发说明

更新时间：2026-07-12

## 功能范围

本 MOD 为《杀戮尖塔 2》提供 Live2D 模型管理、显示和配置能力：

- 通过 Godot 文件选择窗口导入 `.model3.json` 及其完整依赖。
- 同时管理和显示多个模型。
- 支持在独立详细页重命名模型，并从模型卡片导出仅包含当前模型的配置包。
- 独立配置主菜单和游戏内场景；地图与战斗共用同一套游戏内配置。
- 单模型按字段继承全局配置；显式值覆盖全局值。
- 在独立预览页拖动位置、缩放和旋转，确认后保存。
- 扫描 Motion/Expression，并通过 RitsuLib 绑定快捷键。
- 使用 `.live2dpack` 导入、导出模型、配置和快捷键。
- 随 MOD 附带 MiSide 普通测试模型文件，不自动导入、不设置默认配置。
- 提供简体中文、日语、英语界面，英语为回退语言。

设置界面拆分为两个独立导航页：

- “模型管理”：导入、配置包和单模型设置。
- “全局配置”：总控快捷键、主菜单/游戏内默认值、播放与渲染默认值。

模型管理页只显示模型摘要和常用操作。点击“详细设置”后进入 RitsuLib 的独立隐藏子页面，通过主菜单、
游戏内、快捷键三个标签页编辑该模型；该子页面仅在激活时显示，不会长期占用设置侧栏。
全局配置页使用 RitsuLib 原生可折叠 section。

## 默认行为

- 主菜单全局默认布局参考 MiSide MOD：在 1920×1080 下等价于模型原点 `(1420.544, 1696.936)`、
  缩放 `0.5`；配置中使用右下锚点和偏移 `(-499.456, 616.936)`，以继续适配其他分辨率。
- 打开主菜单子页面或模态窗口时，主菜单模型自动隐藏。
- 主菜单原版“设置”下方提供“Live2D 设置”入口。
- 全局快捷键是所有 Live2D 模型唯一的启用/关闭总开关，不修改模型的场景配置。

MiSide 仅作为随包测试素材部署到 MOD 的 `resources/MiSide.zip`。由于游戏会把 `mods` 下的任意 JSON 递归识别成
MOD 清单，模型装在普通 ZIP 中；解压到 `mods` 目录之外后即可像其他模型一样导入。它不会自动导入、
不会出现在特殊设置区域，也没有特殊的场景可见性或布局逻辑。

## 配置规则

运行值按以下顺序解析：

```text
程序默认值 → 全局配置 → 单模型非空覆盖值
```

单模型覆盖字段使用可空类型：

- `null`：继承全局配置。
- 非 `null`：使用模型自己的值。
- 清除覆盖值：恢复继承。

继承解析只由 `Live2DConfigResolver` 完成。运行时、设置界面、预览页和配置包不得自行实现另一套规则。

配置中不提供全局或单模型 `Enabled` 字段。全局快捷键的“全部关闭”是运行时状态；再次触发会恢复
各模型原有的场景可见性，不写入任何场景字段。单模型仍可配置在哪些场景显示，但不能绕过全局快捷键状态。

## 预览编辑器

每个模型的“预览并调整”按钮打开独立全屏编辑页：

- 切换主菜单、游戏内目标场景。
- 左键拖动画布调整位置。
- 滚轮或滑杆调整缩放。
- `Shift + 滚轮`或滑杆调整旋转。
- 数值框精确调整水平、垂直偏移。
- 可模拟当前窗口、1080p、1440p、超宽屏和 4K 分辨率；非等比例画布使用留黑预览。
- “保存修改”写入草稿中实际变更的字段。
- “取消”或 `Esc` 放弃所有草稿。

未调整字段不会被固化，继续继承全局配置。

位置偏移以 `1920×1080` 为参考坐标保存。运行时根据实际视口的较短缩放轴等比换算位置和模型大小，
锚点始终取实际视口边界或中心。因此 16:9、超宽屏和其他纵横比下模型不会被拉伸，窗口尺寸变化后
运行时会自动重建模型布局。预览拖动会先从画布坐标转换为目标分辨率坐标，再转换回参考坐标。

## 运行时结构

```text
RitsuLib 配置存储
    ↓
Live2DConfigResolver
    ↓
Live2DRuntimeManager
    ├─ MainMenu host
    ├─ Map host
    └─ Combat host
          ↓
    Live2DModelInstance
          ↓
      gd_cubism
```

所有游戏方法补丁均通过 RitsuLib 的 `CreatePatcher`、`IPatchMethod` 和
`ApplyRequiredPatcher` 注册。禁止直接创建 Harmony 实例或调用 `PatchAll`。

当前补丁负责：

- 主菜单、地图、战斗宿主挂载；后两者统一解析为游戏内配置。
- 主菜单子页面显示状态同步。
- 模态窗口打开、关闭时同步主菜单模型显示。

## 模型与配置包

导入器会校验并复制 `.moc3`、纹理、物理、姿势、动作和表情资源到受管理目录。
路径穿越、缺失文件和无效入口会被拒绝。

启动时会核对配置记录对应的受管理 `.model3.json`。入口文件被手动删除、模型目录不存在或路径无效时，
自动删除失效配置并重新排列模型顺序；打开模型管理页时会再次检查，以同步游戏运行期间的外部文件变更。

`.live2dpack` 使用 ZIP，包含版本化清单、SHA-256、模型资源和设置。导入时防护：

- ZIP 路径穿越和符号链接。
- 重复条目、异常大小和异常压缩比。
- 配置版本过新。
- 内容哈希重复和模型 UUID 冲突。

开发测试素材位于 `examples/`，随 MOD 发布的附件只有 `resources/MiSide.zip`。该目录没有专用
导入流程；需要测试时，将 ZIP 解压到 `mods` 目录之外，再像其他模型一样选择其中的
`.model3.json` 导入。

## PCK 与 Shader

gd_cubism 原生渲染器通过固定 `res://addons/gd_cubism/res/shader/*` 路径加载 10 个 Shader。
这些资源必须包含在 `Live2D.pck` 中，`Live2D.json` 必须保持 `has_pck: true`。

初始化会逐项加载 10 个 Shader。验证失败时不会注册场景补丁，以避免模型节点存在但显示纯白。

正式导出、验证、构建和部署：

```powershell
.\Tools\export-live2d-pck.ps1 -Configuration Release
```

脚本使用 Godot 4.5.1 Mono 导出 PCK，再由隔离工程验证 Shader `10/10`，最后构建并部署 DLL、
PCK、清单、GDExtension 和测试模型。

## 测试

```powershell
dotnet build .\Live2D.csproj -c Debug
dotnet run --project .\Tests\Live2D.SmokeTests.csproj -c Debug
```

冒烟测试覆盖：

- MiSide 模型清单解析。
- 全局继承、单模型覆盖和恢复继承。
- 中日英词条及格式占位符一致性。
- 配置包往返和恶意路径拒绝。

## 目录

```text
Scripts/Configuration/  配置模型、存储、继承解析
Scripts/Models/         模型解析与资源仓库
Scripts/Packs/          普通配置包导入与导出
Scripts/Runtime/        Cubism、场景宿主、模型实例、快捷键
Scripts/UI/             设置页、主菜单入口、预览页、本地化
Live2D/localization/    eng / jpn / zhs
examples/               测试模型源文件
Tests/                  无游戏进程冒烟测试
Tools/                  PCK 导出与验证工具
resources/              随 MOD 发布的普通资源附件
```

## 当前限制

- 发布目标为 Windows x86_64。
- gd_cubism C# 上游包装代码仍有可空引用警告，但项目代码构建无错误。
- 多模型高分辨率纹理和蒙版可能增加显存占用，应通过缩放和蒙版尺寸配置控制。
