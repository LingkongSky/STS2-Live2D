# 模型管理

“模型管理”页面用于导入、整理和删除玩家模型。每个模型有独立的详细设置，但未覆盖的字段会继续
继承全局配置。

## 支持的导入入口

- 单模型：选择 `.model3.json`。
- 配置包：选择 `.live2dpack` 或 `.livepck`。
- 全局配置：导入只包含全局设置的配置包。

单模型导入会检查 model3 中引用的纹理、Motion、Expression、Physics 和 Pose。缺少依赖、无效路径或
路径穿越会直接拒绝导入。

## 全局值与模型覆盖

实际值按以下顺序解析：

```text
程序默认值 → 全局配置 → 单模型覆盖
```

模型字段显示“继承”时，它会随全局配置变化；改成自定义值后只影响当前模型。清除覆盖即可恢复继承。
滤镜和画布蒙版按完整对象覆盖，不逐个子字段混合。

## 主菜单与游戏内

- “主菜单”只控制主菜单宿主中的模型。
- “游戏内”同时用于地图和战斗。
- 全局显示快捷键只临时关闭或恢复所有模型，不会改写各模型配置。

## 重命名与删除

重命名只改变设置界面的显示名称，不改变模型稳定 ID。删除会移除受管理模型及其配置；由其他 Mod
只读注册的 Pack 实例不属于玩家模型库，不会显示在这里。

## 配置包

从模型卡片导出时只包含当前模型及其覆盖值。导入包会根据内容哈希跳过重复模型；只有持久导入才会
写入玩家模型库，其他 Mod 通过 API 进行的只读注册不会改动这里的内容。
